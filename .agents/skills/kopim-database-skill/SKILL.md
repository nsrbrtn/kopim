# Kopim Database Skill

Этот навык регулирует изменения в локальной реляционной базе данных (Drift SQLite), написание миграций, обновление схем и поддержку целостности данных (foreign keys, integrity checks).

## 1. Когда использовать этот навык
Активируйте этот навык при выполнении задач, которые затрагивают:
- Создание или изменение таблиц баз данных (Drift Tables).
- Изменение схемы БД (добавление колонок, индексов, триггеров).
- Написание и тестирование скриптов миграции локальной БД.
- Работу с DAO (Data Access Objects) и Drift-запросами.
- Исправление ошибок целостности БД (foreign key violations).

## 2. Какие файлы читать в первую очередь
- [database.dart](file:///home/artem/StudioProjects/kopim/lib/core/data/database.dart) — центральный конфигуратор Drift БД.
- Папка [drift_schemas/](file:///home/artem/StudioProjects/kopim/drift_schemas) — исторические снимки схем БД для тестирования миграций.
- Модели и DAO в `data/` слоях соответствующих фич (например, `lib/features/transactions/data/`).

## 3. Запрещенные действия
- **Запрещено** вручную изменять автогенерируемые файлы базы данных (`*.g.dart`, `*.drift.dart`).
- **Запрещено** вносить любые изменения в схему БД (включая добавление/удаление таблиц, колонок, индексов) без предварительного ExecPlan, увеличения версии `schemaVersion` в `database.dart`, написания миграции с логикой отката и обязательного написания тестов регрессии.
- **Запрещено** отключать проверки Foreign Keys (`PRAGMA foreign_keys = ON;`) во время транзакций, кроме исключительных случаев в миграциях, явно согласованных в ExecPlan.
- **Запрещено** выполнять тяжелые блокирующие SQL-запросы на UI-изоляте (основном потоке).

## 4. Какие проверки запускать
- **Обязательный запуск локальных целевых тестов (scoped tests):** Запустите тесты затронутых DAO или сущностей БД через `./tool/ai_check.sh <path_to_test>`.
- **Прогон регрессионных тестов:** Запустить тесты миграции и целостности БД: `flutter test test/integration/db/` (или аналогичный путь к тестам БД).
- Сгенерировать код БД: `dart run build_runner build --delete-conflicting-outputs`.
- Прогнать базовый анализ проекта: `./tool/ai_check.sh`.


## 5. Типичные ошибки для предотвращения
- **Забытая миграция при изменении таблицы:** Если добавить колонку в класс таблицы Drift, но не написать шаг миграции в `database.dart`, приложение упадет у пользователей при обновлении из-за отсутствия колонки в SQLite.
- **Ошибки обратной совместимости при импорте:** Восстановление старых бэкапов (экспортированных в старых версиях БД) на новых схемах должно обрабатываться корректно без падений.
- **Игнорирование Foreign Key ограничений:** Попытка вставить запись со ссылкой на несуществующий ID родительской сущности (например, транзакцию с несуществующей категорией).

## 6. Ожидаемый результат (Expected Output)
- Новые/измененные таблицы успешно сгенерированы через `build_runner`.
- Написана миграция с логикой наката и планом отката.
- Обновлен файл `database.dart` (версия схемы увеличена).
- Добавлен новый снимок схемы в `drift_schemas/` с помощью генератора Drift (если применимо).
- Написаны тесты, проверяющие переход схемы со старой версии на новую с сохранением данных.
