# ExecPlan: Обновление экрана «Категории и тэги»

## Context and Orientation
- Цель: изменить UX экрана управления категориями по запросу: новый заголовок, плоский список категорий/подкатегорий без раскрытия и свайпов, редактирование по тапу, удаление из экрана редактирования.
- Область кода: `lib/features/categories/presentation/screens/manage_categories_screen.dart`, `lib/l10n/app_ru.arb`, `lib/l10n/app_en.arb`, с последующей генерацией l10n.
- Контекст/ограничения: не редактировать автоген вручную; сохранить существующие сценарии создания/редактирования категории и тэгов.
- Риски: визуальные регрессии списка категорий и дублирование удаления (свайп + кнопка) при неполном удалении старого поведения.

## Interfaces and Dependencies
- Внешние сервисы (Firebase/Analytics/Crashlytics): нет.
- Локальные зависимости (Drift, codegen): генерация локализаций `flutter gen-l10n`.
- Затрагиваемые API/модули: UI экрана `ManageCategoriesScreen`, bottom sheet редактора категории.

## Plan of Work
- Удалить swipe/expand механику и заменить на плоский иерархический список с тапом для редактирования.
- Добавить кнопку удаления в экран редактирования категории/подкатегории.
- Обновить заголовок экрана в локализации и прогнать проверки.

## Concrete Steps
1) Переработать `manage_categories_screen.dart`: убрать `Dismissible`, анимацию подсказки свайпа и expandable карточки; внедрить плоский список родительских и дочерних категорий.
2) Добавить удаление из `_CategoryEditorSheet` для существующей категории/подкатегории.
3) Обновить `profileManageCategoriesTitle` в `app_ru.arb` и `app_en.arb`, сгенерировать l10n.
4) Запустить `dart format --set-exit-if-changed .` и `flutter analyze`.

## Validation and Acceptance
- Команды проверки:
```bash
dart format --set-exit-if-changed .
flutter analyze
```
- Acceptance criteria:
  - Заголовок экрана: «Категории и тэги».
  - На вкладке категорий виден единый список: родительские категории и их подкатегории сразу, без раскрытия.
  - Подкатегории визуально меньше и расположены ниже/правее родителя.
  - Свайп влево/вправо по категориям отсутствует.
  - Тап по категории/подкатегории открывает редактирование.
  - На экране редактирования есть кнопка удаления для существующей категории/подкатегории.

## Idempotence and Recovery
- Что можно безопасно перезапускать: форматирование, анализ, генерация l10n.
- Как откатиться/восстановиться: откатить изменения в целевых файлах экрана и локализаций.
- План rollback (для миграций): не требуется.
