# ExecPlan: Обновление виджета операций по категориям в аналитике

## Context and Orientation
- Цель: обновить виджет категорий в аналитике: новый заголовок, переключатель доходы/расходы, опциональное включение переводов и кредитных платежей в график и список.
- Область кода: `lib/features/analytics/presentation/analytics_screen.dart`, `lib/features/analytics/presentation/controllers/analytics_providers.dart`, `lib/features/analytics/presentation/widgets/analytics_chart.dart`, `lib/l10n/*`, тесты аналитики.
- Контекст/ограничения: сохранить текущую архитектуру Riverpod + immutable модели, не трогать автоген вручную.
- Риски: неконсистентная агрегация переводов/кредитных операций, лишние rebuild в UI.

## Interfaces and Dependencies
- Внешние сервисы (Firebase/Analytics/Crashlytics): нет.
- Локальные зависимости (Drift, codegen): Riverpod провайдеры, l10n генерация при изменении ARB.
- Затрагиваемые API/модули: аналитические провайдеры и UI экрана аналитики.

## Plan of Work
- Добавить состояние/переключатели для типа операций и опции показа переводов/кредитных платежей.
- Расширить источник данных для виджета категорий и собрать итоговый набор элементов графика.
- Обновить UI (заголовок, переключатель, чип), покрыть тестами и прогнать проверки.

## Concrete Steps
1) Добавить провайдеры/модель для агрегирования переводов и кредитных платежей в виде категориальных элементов.
2) Обновить `_TopCategoriesPager`/`_TopCategoriesPage`: заголовок, переключатель «Доходы/Расходы», чип «Показывать переводы».
3) Интегрировать новые данные в chart/list, обновить/добавить тесты и документацию при изменении поведения.

## Validation and Acceptance
- Команды проверки:
```bash
dart format --set-exit-if-changed .
flutter analyze
flutter test --reporter expanded
```
- Acceptance criteria:
  - Заголовок виджета: «Операции по категориям».
  - Есть переключатель доходы/расходы над графиком.
  - При включенном чипе «Показывать переводы» в график добавляются переводы и кредитные платежи.
  - При выключенном чипе переводы и кредитные платежи исключаются.

## Idempotence and Recovery
- Что можно безопасно перезапускать: форматирование, analyze, тесты.
- Как откатиться/восстановиться: откатить изменения в `analytics_screen.dart` и связанных провайдерах.
- План rollback (для миграций): не требуется.

## Surprises & Discoveries
- Для добавления нового `@riverpod`-провайдера потребовался бы `build_runner`; для ускорения внедрен обычный `StreamProvider` без генерации.

## Decision Log
- Решение: использовать существующий `analyticsCreditDebtOperationsProvider` для кредитных платежей и отдельную агрегацию переводов из транзакций текущего окна.
