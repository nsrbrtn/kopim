# ExecPlan: Обновление блока "Траты по категориям" в аналитике

## Context and Orientation
- Цель: привести экран аналитики к макету для вкладки "Траты по категориям" и синхронизировать интерактивное поведение графика/чипа категории.
- Область кода: `lib/features/analytics/presentation/analytics_screen.dart`.
- Контекст/ограничения: использовать только тему/токены и данные категорий; без хардкода цветов/шрифтов; не менять автогенерируемые файлы.
- Риски: визуальные расхождения с макетом, регресс интерактивности фильтрации графика, перепутанная логика выбора категории и раскрытия вложенных категорий.

## Interfaces and Dependencies
- Внешние сервисы (Firebase/Analytics/Crashlytics): нет.
- Локальные зависимости (Drift, codegen): не требуются.
- Затрагиваемые API/модули: виджеты аналитики, локализация `AppLocalizations`, цветовые утилиты категорий.

## Plan of Work
- Завершить перестройку UI блока "Траты по категориям" по макету.
- Исправить поведение выбора категории только по тапу на графике и фильтрацию данных графика по выбранной категории.
- Проверить и стабилизировать отображение чипов категорий, включая обрезку длинных названий.

## Concrete Steps
1) Довести layout и spacing/typography верхнего блока и контролов под графиком до поведения как в макете.
2) Исправить логику `_highlightKey`: появление активного чипа только при выборе сегмента/бара на графике; сброс фильтра только по крестику.
3) Проверить `displayItems/displayTotal` и legend, чтобы при выборе категории график показывал отфильтрованные данные выбранной категории.
4) Прогнать форматирование и анализатор, обновить прогресс в плане.

## Validation and Acceptance
- Команды проверки:
```bash
dart format --set-exit-if-changed lib/features/analytics/presentation/analytics_screen.dart
flutter analyze lib/features/analytics/presentation/analytics_screen.dart
```
- Acceptance criteria:
  - Верхние вкладки и блок "Траты по категориям" визуально соответствуют макету.
  - Активный чип категории появляется только после тапа по графику.
  - По выбранной категории график и суммы фильтруются.
  - Сброс фильтра работает только по нажатию на крестик в активном чипе.
  - Длинные названия категорий в чипах обрезаются и не ломают layout.

## Idempotence and Recovery
- Что можно безопасно перезапускать: `dart format`, `flutter analyze`.
- Как откатиться/восстановиться: вернуть изменения в `analytics_screen.dart`.
- План rollback (для миграций): не требуется.
