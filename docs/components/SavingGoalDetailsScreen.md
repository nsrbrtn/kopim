# SavingGoalDetailsScreen

## Назначение

Экран деталей одной копилки (`SavingGoalDetailsScreen`) показывает прогресс накопления, прогноз достижения цели, связанные счета хранения, аналитику вкладов и историю пополнений.

После обновления UX экран визуально следует референсу detail-view:
- крупные секционные карточки с мягким градиентом из темы;
- акцентный блок прогресса с двумя метриками;
- отдельная карточка прогноза;
- аналитика в виде donut chart + summary panel;
- широкая нижняя CTA-кнопка пополнения.

## Что изменилось

- Перекомпонован экран деталей копилки под более выразительную mobile-first иерархию.
- Вместо базовых `Card` используются кастомные секционные контейнеры на `ColorScheme`.
- CTA "Пополнить" перенесён в широкий нижний action, ближе к визуальному образцу.
- История вкладов и список счетов получили более заметные tile-контейнеры.

## Зачем

- Улучшить визуальное качество и читаемость экрана накопления.
- Сделать экран ближе по восприятию к premium/mobile-detail интерфейсу.
- Сохранить существующую бизнес-логику без изменения провайдеров и контрактов данных.

## Как проверить

```bash
flutter analyze lib/features/savings/presentation/screens/saving_goal_details_screen.dart
flutter test test/features/savings/presentation/screens/saving_goal_details_screen_test.dart
```

Ручная проверка:
- открыть любую копилку;
- проверить секции `Прогресс`, `Прогноз накопления`, `Счета накопления`, `Аналитика по счёту`, `История вкладов`;
- убедиться, что нижняя кнопка `Пополнить` остаётся доступной и не перекрывает контент;
- проверить экран в светлой и тёмной теме.

## Breaking changes

Нет. Изменения касаются только визуальной композиции экрана.
