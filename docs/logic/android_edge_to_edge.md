# Android edge-to-edge

## Контекст

Начиная с Android 15 для приложений с `targetSdk` 35 отображение от края до края включается по умолчанию. Системные бары становятся прозрачными или полупрозрачными, а контент приложения может рисоваться под статус-баром, navigation bar и вырезами экрана.

## Правила проекта

- Не задавать цвета системных баров через `Window.setStatusBarColor`, `Window.setNavigationBarColor`, `Window.setNavigationBarDividerColor` и Flutter-обертки, которые приводят к тем же Android API.
- Edge-to-edge включается при старте Android activity через `WindowCompat.enableEdgeToEdge(window)`.
- На Flutter-стороне стартовый режим системного UI задается через `SystemUiMode.edgeToEdge`.
- Интерактивные элементы у краев экрана должны учитывать inset'ы через `SafeArea`, `MediaQuery.padding` или компоненты Material, которые уже обрабатывают системные отступы.
- Нижняя навигация должна оставаться внутри bottom safe area, чтобы gesture bar и трехкнопочная навигация не перекрывали кнопки.

## Что проверять перед релизом

- Android 15+ с gesture navigation: контент не закрывает статус-бар, gesture handle не перекрывает нижнюю навигацию и FAB.
- Android 15+ с трехкнопочной навигацией: нижние кнопки и списки не попадают под navigation bar.
- Android 14 и ниже: визуально нет регрессий после `enableEdgeToEdge`.
- Экраны входа, онбординга, настроек, модальные формы и пустые состояния тоже проверяются, потому что они реже проходят ручное тестирование.
