# Мини-онбординг первого запуска

## Что изменилось

После чистой установки на `Home` появляется компактный чеклист первого запуска. Он не заменяет основной интерфейс и не уводит пользователя в отдельную последовательность полноэкранных экранов. Чеклист ведет по базовому сценарию:

1. Создать первый счет.
2. Добавить категорию.
3. Записать первую транзакцию.
4. Заполнить профиль.
5. Создать цель накопления.

Карточка автоматически отмечает выполненные шаги по реальным данным пользователя и исчезает с `Home`, когда все обязательные шаги завершены. Если пользователь скрывает карточку, ее можно открыть снова из `Меню`.

## Зачем

- Дать понятный старт после установки без перегрузки классическим onboarding carousel.
- Сохранить фокус на реальных действиях в приложении, а не на объясняющих слайдах.
- Не дублировать бизнес-данные: прогресс считается по существующим счетам, категориям, транзакциям, профилю, целям и бюджетам.
- Не показывать ложный first-run onboarding существующим пользователям после обновления.

## Как это работает

- Локальное состояние показа хранится в `SharedPreferences`.
- Мини-онбординг автоматически активируется только если приложение видит действительно пустой стартовый state.
- Если у пользователя уже есть meaningful data, чеклист сам не показывается после обновления приложения.
- Шаг `Добавить категорию` считается выполненным только для пользовательских категорий, а не для системных.
- Для эвристики existing user учитываются и бюджеты, чтобы не показывать ложный пустой старт пользователю, который уже ведет учет через budget-first сценарий.

## Как проверить

1. Установить приложение на чистое окружение.
2. Открыть `Home` и убедиться, что виден чеклист `Что сделать дальше`.
3. Последовательно создать счет, категорию, транзакцию, заполнить профиль и создать цель.
4. Проверить, что шаги отмечаются автоматически, а заблокированные шаги открываются только по порядку.
5. Скрыть карточку и убедиться, что она пропадает с `Home`.
6. Открыть `Меню -> План первого запуска` и убедиться, что чеклист доступен повторно.
7. На профиле existing user / в тестовом состоянии с уже созданными данными убедиться, что карточка не появляется автоматически после старта.
