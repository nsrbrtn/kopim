# Web fallback и ручная выгрузка данных

## Назначение

В некоторых браузерах Firebase может быть недоступен (например, из-за ограничений хранилищ).
Чтобы приложение оставалось рабочим, используется мягкий web‑fallback: локальная работа без
авторизации и синхронизации.

## Поведение

- Если Firebase недоступен на web, приложение запускается в офлайн‑режиме.
- Авторизация и синхронизация отключаются.
- Данные продолжают сохраняться локально (Drift).
- В UI показывается предупреждение и бейдж «Офлайн».

## Ручная выгрузка данных

При работе в офлайн‑режиме есть риск потери данных (например, при очистке данных браузера).
Пользователю нужно сделать выгрузку данных через настройки приложения и сохранить резервную копию.
