# Web fallback и ручная выгрузка данных

## Назначение

В некоторых браузерах Firebase может быть недоступен (например, из-за ограничений хранилищ).
Чтобы приложение оставалось рабочим, используется мягкий web‑fallback: локальная работа без
авторизации и синхронизации.

## Поведение

- Если Firebase недоступен на web, приложение запускается в офлайн‑режиме.
- Авторизация и синхронизация отключаются.
- Данные продолжают сохраняться локально (Drift).
- В UI показывается предупреждение и бейдж «Офлайн».

## Ручная выгрузка данных

При работе в офлайн‑режиме есть риск потери данных (например, при очистке данных браузера).
Пользователю нужно сделать выгрузку данных через настройки приложения и сохранить резервную копию.

## Что происходит после очистки браузера

- Web-версия хранит локальные данные в браузерном storage через Drift.
- Если пользователь очистит site data, browser data, local storage, IndexedDB или удалит данные PWA, локальная база будет удалена.
- После такой очистки офлайн-данные без синхронизации восстановить нельзя.

Практический вывод:

- офлайн web защищает от временного отсутствия сети, но не является надежным долговременным хранилищем;
- для защиты данных нужен хотя бы один из двух механизмов:
  - рабочая Firebase-синхронизация с учетной записью;
  - регулярный ручной экспорт резервной копии.

## Как снизить риск потери данных

- По возможности держать Firebase доступным и синхронизацию включенной.
- Для браузеров с fallback-режимом показывать предупреждение о локальном хранении и риске очистки данных.
- Перед очисткой браузера, сменой устройства или переустановкой PWA делать экспорт данных.
- Не обещать пользователю, что web-offline переживет очистку браузера: это технически не гарантируется приложением.
