# Форма бюджета (Budget Form)

## Что изменилось
- В режиме `Выбранные категории` лимиты по категориям стали иерархическими: лимит родителя включает лимиты подкатегорий.
- В секции `Лимиты по категориям` выбранные категории отображаются деревом, чтобы parent/child-структура была видна при редактировании.
- Для выбранной категории без родителя с лимитом теперь обязателен собственный лимит.
- Подкатегория может не иметь собственного лимита, если она остается внутри лимита ближайшей родительской категории.
- Сумма бюджета `byCategory` теперь считается как сумма allocation-корней, а не сумма всех заполненных полей.

## Зачем
- Убрать двойной учет лимитов, когда у бюджета есть и родительская категория, и ее подкатегории.
- Сделать поведение формы предсказуемым для сценария `Категория = 10000`, `Подкатегории = 3000`, где direct-лимит родителя остается `7000`.
- Сохранить иерархию категорий в UI формы, чтобы ограничения было проще проверять до сохранения.

## Как проверить
- Открыть создание/редактирование бюджета с областью `Выбранные категории`.
- Выбрать родительскую категорию и убедиться, что дочерние категории автоматически включаются в форму и отображаются в секции `Лимиты по категориям` с отступами.
- Задать лимит родителю `10000`, одной или нескольким подкатегориям суммарно `3000` и сохранить бюджет.
- Проверить, что бюджет сохраняется с общей суммой `10000`, а не `13000`.
- Попробовать задать подкатегориям суммарный лимит больше лимита родителя и убедиться, что форма не сохраняется.
- Попробовать выбрать верхнеуровневую категорию без собственного лимита и убедиться, что форма не сохраняется.

## Breaking changes
- Нет на уровне схемы данных, но логика валидации и подсчета лимитов для `byCategory` стала stricter и перешла на inclusive-модель.
