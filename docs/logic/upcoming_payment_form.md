# Форма предстоящего платежа

## Назначение
Экран добавления и редактирования предстоящего платежа. Отвечает за ввод параметров повторяющихся платежей, настройку напоминаний и автопроведения.

## Структура формы
- **Контейнер 1:** название, счёт, тип движения (`Расход`/`Доход`), категория, сумма.
- **Контейнер 2:** день месяца, переключатель `Напомнить`; при включении отображаются время уведомления, «за сколько дней напомнить» и переключатель автопроведения.
- **Контейнер 3:** сворачиваемый блок заметки (в заголовке только название и стрелка; поле ввода появляется после раскрытия).

Контейнеры используют фон `surfaceContainer`, поля ввода — `surfaceContainerHigh` для контраста.

## Тип движения и категории
- Поле `Тип` определяет финансовое направление правила: `Расход` или `Доход`.
- При выборе `Расход` в списке категорий показываются только категории типа `expense`.
- При выборе `Доход` — только категории типа `income`.
- Если у выбранного типа нет категорий, используется полный список, чтобы не блокировать сохранение.

## Выбор даты и времени
- День месяца выбирается через календарь (`showDatePicker`). Сохраняется только число дня.
- Время уведомления выбирается через `showTimePicker` и доступно только при включенном `Напомнить`.
- Для обоих действий отображаются иконки календаря и часов в поле ввода.

## Логика переключателей
- `Напомнить` по умолчанию выключен для нового предстоящего платежа.
- При выключенном `Напомнить` поля времени/дней скрыты.
- `Проводить автоматически` по умолчанию выключен и показывается только при включенном `Напомнить` под полем «за сколько дней напомнить».

## Удаление платежа
- Кнопка удаления отображается **только** в режиме редактирования.
- При удалении показывается подтверждающий диалог.
- После успешного удаления экран закрывается, уведомления отменяются.

## Как проверить
1) Открыть создание предстоящего платежа — форма разделена на 3 контейнера.
2) Выбрать день месяца через календарь и время уведомления через тайм-пикер.
3) Открыть редактирование платежа — доступна кнопка удаления рядом с сохранением.
