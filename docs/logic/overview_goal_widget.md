# Виджет «Цель» на Overview

## Что изменилось

- Карточка «Цель» на `OverviewScreen` переведена со статического состояния на реальные данные из `saving_goals`.
- Для активной карточки выбирается фокусная цель (с минимальным оставшимся остатком до цели, при равенстве — более свежая по `updatedAt`).
- Кнопка «Пополнить» ведет в `ContributeScreen` выбранной цели.

## Модель данных

- `saving_goals` содержит связь `account_id` на счет накоплений.
- Для существующих целей миграция v40 создает скрытый счет накоплений и выполняет backfill `account_id`.

## Пополнение

- Пополнение цели выполняется как перевод:
  - `transaction.type = transfer`
  - `accountId = sourceAccountId`
  - `transferAccountId = savingGoal.accountId`
  - `savingGoalId = savingGoal.id`
- Сумма перевода равна фактическому приросту цели (`appliedDelta`), с учетом cap до `targetAmount`.

## Состояния карточки

- `data`: имя цели, процент прогресса, оставшаяся сумма и действие «Пополнить».
- `empty`: приглашение создать цель, действие «Создать цель».
- `loading`: плейсхолдер текста/прогресса.
- `error`: fallback как `empty`.

## Как проверить

1. Создать новую цель накоплений.
2. Убедиться, что у цели появился связанный скрытый счет накоплений.
3. Нажать «Пополнить», выбрать исходный счет и внести сумму.
4. Проверить, что создана транзакция `transfer`, баланс source уменьшился, баланс счета цели увеличился, а карточка прогресса обновилась.
